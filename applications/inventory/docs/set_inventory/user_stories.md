# User Stories
Outlines the intent of end users and should be used to guide API and frontend design.

## Inventory Tracking:
- As a user, I want to take a new set and add it to my inventory via scanning its barcode.
- As a user, I want to take a new set, one that I already own, and add it to my inventory, via its barcode, so I know I have multiple boxes of that set.
- As a user, I want to remove a set from my inventory in the case that I sell it via barcode.
- As a user, I want to update the information for a set I have to reflect that I have built/opened the set via barcode.

## Inventory Validation:
- As a user, I want to scan LEGO boxes in bulk to validate that all boxes are accounted for that are in my inventory.
- As a user, I want to scan LEGO boxes in a storage location and validate that all expected LEGO sets are present for that storage location.

## Inventory/Storage Management
- As a user, I want to scan a set and update its storage location.
- As a user, I want to create a new storage location with a description or barcode.
- As a user, I want to rename an existing storage location with a description or barcode.
- As a user, I want to safely delete a storage location knowing that no LEGO boxes are stored there.
